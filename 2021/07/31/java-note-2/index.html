面向对象编程OOP

Obeject-Oriented Programming

以类的方式组织代码，以对象的组织（封装）数据

三大特性

  • 封装
  • 传承
  • 多态

定义类

封装好处

  1. 提高安全性，保护数据
  2. 隐蔽代码的实现细节
  3. 统一接口
  4. 系统可维护增加了

继承

方法重写

重写都是方法的重写和属性无关

静态方法和非静态方法区别很大，非静态才可以重写

  1. 方法名必须相同
  2. 参数列表必须相同
  3. 修饰符，范围可以扩大，但不可以缩小。public>protected>default>private
  4. 抛出的异常，范围可以缩小，但不能扩大

多态

父类的引用指向子类，但不能调用子类的方法

对象能执行哪些方法，主要看对象左边的类型

接口

关键字 interface

作用

  1. 约束
  2. 定义方法，让不同的人实现
  3. 接口不能被实例化，接口中没有构造方法
  4. 可以实现多个接口
